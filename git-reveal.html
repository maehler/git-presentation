Version control software

  • Initially created for software development
  • Some of the most used ones are
    • Subversion (SVN)
    • Mercurial
    • Git

Purpose

  • Enable easier collaboration
  • Keep a history for a project
  • Backup (kind of)
  • Good for plain text files such as source code
  • Less good for binary formats such as images or Word documents

How does Git work?

  • Command line tool
  • Various graphical user interfaces available
  • I will focus on the command line tool
The graphical user interfaces can be of great help, but they are of even greater help if you already understand the command line tool.

Definitions

Repository
Collection of files and their history
Workspace
Where the work is being done
Commit
A snapshot of the workspace

Distributed

  • Every user has its own copy of the workspace
  • Almost everything is done locally
  • No internet connection needed

Commands

  • git init: initialise a repository
  • git status: check the status of the repository
  • git add: add changes (stage)
  • git commit: commit the change to the repository
  • git push: push changes to remote repository
  • git pull: get changes from remote repository
There are many more commands available, but these are perhaps the ones that are used more regularly

Branching

  • git branch: create a new branch
  • git merge: join branches
  • git checkout: move to commit or branch
Commit history can be seen as a tree structure, and trees have branches.

Why do I use Git?

  • Helps me structure my work

How can Git help me structure my work?

  • Have to think before I write
  • Divide work into smaller chunks
  • Commit smaller, "self-contained" changes
